## 1. One-paragraph summary

DD v2 §11.6 proposed a frozen Gemma 4 E2B base + a small confidence-gated sidecar that reads the model's last-layer hidden state at the final response token (`hs_last`) and gates whether to assert or hedge. The theory was that this hidden state encodes a correctness-correlated direction the sidecar can exploit without any retraining of the base. A supervised CV ceiling at n=100 on step16 looked promising (AUC 0.68, balanced assertion asymmetry −0.34). On 193 fresh Chef prompts drawn from the same distribution, judged with the same rubric, and using the same model under the identical Unsloth 4-bit load, **every part of the theory fails**. The `hs_last` probe collapses to chance on fresh data (AUC 0.52). Extending the probe to per-axis labels across all 4 in-scope Chef dispositions (Humble / Deliberate / Adversarial-self / Persistent) plus the factual anchor produces 0/4 passing axes, with three transfer AUCs below chance. A multi-layer variant (mean of the last 4 hidden states) recovers some signal but still falls under the behavioral bar (asym −0.13 vs −0.15). A cross-model replication on SmolLM2-1.7B-Instruct fails the same gate (AUC 0.519) through a different failure route — a fresh-prompt capability cliff (pos rate 0.56 → 0.047) that leaves the probe no positive class to separate. Two frozen bases, two failure modes, one conclusion: at sub-2B instruct scale on judge-based domains the §11.6 surface does not carry the signal it was assumed to carry. The branch closes.

## 2. Context

**Where §11.6 sat in DD v2.** After §7 falsified B.1/B.2 attention-head tempering and imitation-into-weights closed at sub-2B scale across multiple dispositions, §11.6 was opened as a *scoped, cheap, frozen-base alternative*: if you cannot move disposition via weights, maybe you can read confidence from the residual stream and gate output behavior at inference time without touching the base at all.

**What we agreed to measure before committing GRPO.** Per user bar locked 2026-04-09: §11.6 counts as a "successful DD v2 pivot" only if all 4 in-scope Chef dispositions (Humble, Deliberate, Adversarial-self, Persistent) can be both **measured** and **shown to have hs_last signal**, with the factual anchor holding. If even one of the four cannot be measured or has no signal, STOP — do not spend the $55 GRPO budget. MCAS / UNCERTAIN prompts were explicitly deferred.

## 3. Method

### 3.1 Data

- **Step16 baseline** (training/gate fit): 100 type1 Chef error-correction prompts, Gemma 4 E2B-it greedy responses under Unsloth 4-bit (`dd-v2/eval/results/baseline_gemma4_v2_from_step16.json`). Labels = gold-checklist coverage ≥ 0.5 (the v3 rubric). Positive rate 58/100.
- **Fresh transfer set**: 193 NEW Chef prompts generated by DeepSeek V3.2-exp to match the step16 type1 shape (`dd-v2/eval/results/v1_full_fresh_prompts.json`), gold checklists generated by the same judge (`v1_full_fresh_gold.json`), responses from the SAME Gemma 4 E2B-it load (`v1_full_fresh_responses.json`, batched generation, batch=8, greedy, 512 tokens). Per-claim YES/NO scoring → `label = coverage ≥ 0.5`. Positive rate 55/193 = 0.285.
- **v4 5-axis labels**: both step16 (`judge_v4_step16.json`) and fresh (`judge_v4_fresh.json`) re-scored with `eval_judge_rubric_v4.py` (v2's response-blind 2-pass certainty + v1's full 5-axis rubric; "do not reward verbosity" system prompt). For the per-axis probe: binarize to `high = score ≥ 4`.

### 3.2 Features

All features are extracted under the identical Unsloth `FastModel.from_pretrained("unsloth/gemma-4-E2B-it", load_in_4bit=True)` load that generated the responses, with `FastModel.for_inference(m)`. For each row, the prompt-response sequence is forward-passed once to obtain `output_hidden_states=True`, and features are taken at the **last non-pad position** of the generated portion:

| Feature | Shape | Source |
|---|---|---|
| `hs_last` | [1536] | final layer, last response token |
| `last-4 mean` | [1536] | mean of layers `-4..-1`, last response token |

### 3.3 Probes

Every probe is the identical recipe: `StandardScaler → LogisticRegression(max_iter=2000, C ∈ {1.0, 0.1})`. Within-distribution: stratified 5-fold CV on step16. Cross-distribution: train on full step16, predict on fresh. No hyperparameter search, no regularization sweep beyond the two C values, no feature selection. The intent is to replicate the v1a 4-bit run exactly, then extend it to new axes / new pooling without changing anything else.

### 3.4 Gates

| Test | AUC bar | Asymmetry bar |
|---|---|---|
| v1-full factual | fresh AUC ≥ 0.63 | balanced asym @ p_c ≥ 0.70 ≤ −0.15 |
| Per-axis disposition sweep | cv AUC ≥ 0.65 AND transfer AUC ≥ 0.60 | n/a (ranking not gating) |
| Multi-layer variant | fresh AUC ≥ 0.60 | balanced asym @ p_c ≥ 0.70 ≤ −0.15 |
| Cross-axis separability | all |r| < 0.80 between OOF predictions | — |

## 4. Results

### 4.1 Within-distribution (v1a, already in the approach doc)

| Config | AUC | Balanced asym @ p_c≥0.70 |
|---|---|---|
| hs_last, LogReg C=0.1, 5-fold CV on step16 n=100 | 0.683 | **−0.341** ✅ |
| hs_last, LogReg C=0.1, 5-fold CV on step16 n=100, Unsloth 4-bit | 0.663 | **−0.341** ✅ |

Interpreted at the time as: the sidecar theory has real ceiling headroom. In retrospect: this was fitting intra-distribution noise on a 100-item sample.

### 4.2 Cross-distribution (v1-full, new)

| Config | Fresh AUC | Balanced asym @ p_c ≥ 0.70 | Pass |
|---|---|---|---|
| hs_last, LogReg C=1.0 | 0.509 | +0.008 | ❌ |
| hs_last, LogReg C=0.1 | **0.516** | **−0.043** | ❌ |

AUC is chance. Asymmetry is −0.04 against a bar of −0.15. The v1a ceiling does not transfer.

### 4.3 Per-axis disposition sweep

All 5 axes, hs_last surface, LogReg(C=0.1), n(step16)=99 valid, n(fresh)=192 valid.

| Axis | Disposition | Pos s16/fresh | CV AUC (bar 0.65) | Transfer AUC (bar 0.60) | Pass |
|---|---|---|---|---|---|
| factual_accuracy | capability anchor | 49 / 101 | 0.533 | **0.480** | ❌ |
| hedging_appropriateness | Humble | 79 / 123 | 0.599 | **0.515** | ❌ |
| pedagogical_framing | Deliberate | 81 / 147 | 0.481 | **0.439** | ❌ |
| self_verification | Adversarial-self | 26 / 76 | 0.594 | **0.518** | ❌ |
| completeness | Persistent | 47 / 130 | 0.510 | **0.442** | ❌ |

- **0/4 disposition axes pass.** Factual anchor also fails.
- **3 of 5 transfer AUCs are below 0.50** (factual, pedagogical, completeness). The step16 CV was fitting noise that is actively anti-correlated with fresh labels.
- **Cross-axis OOF correlations:** all |r| < 0.80, 0 flagged. Clean separability — but meaningless once every axis has no signal.

### 4.4 Multi-layer variant

Feature = mean of hidden states at layers −4…−1, last response token. Same LogReg recipe.

| Config | Fresh AUC (bar 0.60) | Balanced asym @ p_c≥0.70 (bar −0.15) | Pass |
|---|---|---|---|
| last-4 mean, C=1.0 | 0.553 | −0.123 | ❌ |
| **last-4 mean, C=0.1** | **0.557** | **−0.130** | ❌ (closest) |
| reference: hs_last C=0.1 | 0.516 | −0.043 | ❌ |

Multi-layer pooling recovers modest signal — asymmetry improves by ~3× — but still misses the bar. The "wrong layer" / "wrong pooling" alibi is closed.

### 4.5 Behavioral spot-check (5 low-score samples per axis)

Counts of items scoring ≤ 2 in the 192 valid fresh responses:

| Axis | Disposition | n (score ≤ 2) | Share |
|---|---|---|---|
| factual_accuracy | anchor | 55 | 29% |
| hedging_appropriateness | Humble | 40 | 21% |
| pedagogical_framing | Deliberate | **12** | 6% |
| self_verification | Adversarial-self | **78** | 41% |
| completeness | Persistent | 36 | 19% |

Observations from the sampled items:

- **Adversarial-self is Gemma's weakest disposition.** Across all 5 sampled low items (and 41% of the fresh set at large), Gemma explains and enumerates causes but does not cross-check ("if X then Y", "this assumes…", "double-check by…"). Real, visible disposition gap.
- **Pedagogical is near-ceiling.** Only 12/192 (6%) low. Gemma is structurally pedagogical by default; no room for a training signal to push it up.
- **Humble failures are "clarifying-question on CERTAIN prompt".** idx=74 (pot-au-feu murky), idx=132 (navets glacés): Gemma asks "I need a little more information" when a canonical answer exists. v4 scores HEDG=1. Frequently paired with FACT=1.
- **Factual misses: domain confusion.** idx=38 "blanched green beans olive drab" → response is about *watering houseplants* ("This is a classic gardening problem!"). FACT=1. Judge caught it cleanly.
- **Wrong-framing pattern.** idx=190 "tarte fine aux pommes not tender" → Gemma replies "Here are a few ways to interpret this statement" as if it were a trivia question. FACT=2, PEDA=1, COMP=2.

**The judge is not the problem.** v4 5-axis scores track visible behavior in the responses, the errors are the ones a human would flag, and the 5 axes are not collinear (item 0 factual=1 pedagogical=5 self_verif=1). If the probe could see what the judge sees, it would learn these axes cleanly. It cannot.

## 5. Interpretation

**Root cause is visible in the samples, not just the metrics.** Gemma 4 E2B-it has a near-single output distribution on Chef error-correction prompts: verbose, structured, confident, pedagogical. Whether the response happens to hit 3/5 canonical facts (label=1) or 1/5 (label=0), the final-token (and last-4-mean) hidden state encodes the same stylistic closure of "verbose confident answer about French cooking," not the truth-value of the claims inside. The behavioral variation that *does* exist (self-verification, hedging calibration, catastrophic domain confusion) lives in the token distribution of the response, not in the final hidden position we were probing.

**Three independent failures rule out the usual escape hatches:**

1. **"The CV was right, you just needed more data."** Refuted: fresh n=193 is twice the step16 n=100, same distribution, same judge, same model, same dtype. The CV ceiling doesn't transfer *at all*.
2. **"You were looking at the wrong axis."** Refuted: all 4 locked disposition axes fail, and the easiest one (factual accuracy, the capability anchor) is the worst offender.
3. **"You were looking at the wrong layer."** Refuted: the multi-layer last-4-mean variant is also under the bar, by ~0.02 AUC.

**What this means for the §11.6 theory.** The theory was: the residual stream at the last response token encodes a correctness-correlated direction that a small confidence-gated head can exploit without retraining the base. Under the 4 tests above, the frozen-base residual stream on Chef at this scale **does not carry that direction** — at least not in any pooled form the standard LogReg probe can read. There is no point in spending $55 on a GRPO-trained sidecar to read a signal that provably is not there.

**What this does NOT mean.** Behavior isn't present (it very much is — see §4.5). It isn't that DD dispositions can't be measured (v4 measures them). It isn't that no signal could ever be recovered (contrastive heads, token-level attention, generative readers are all untested — but spending budget on them is a *different branch*, not a continuation of §11.6). And it doesn't reopen B.1/B.2 head-tempering or imitation-into-weights SFT — those remain falsified under §7.

## 5b. Cross-model replication: SmolLM2-1.7B-Instruct (added 2026-04-09)

To rule out "this is a Gemma instruct-tune artifact," the factual anchor of the §11.6 probe was re-run on a second frozen base: `HuggingFaceTB/SmolLM2-1.7B-Instruct`. Same methodology (step16 features → LogReg → 193 fresh Chef prompts → gold-checklist labels → probe eval). See `FINDING_smollm_baseline_snapshot.md` for full details.

### Results

| Metric                            | Gemma 4 E2B | SmolLM2-1.7B |
|-----------------------------------|-------------|--------------|
| step16 pos rate                   | 0.58        | 0.56         |
| fresh pos rate                    | 0.285       | **0.047**    |
| fresh AUC (best C)                | 0.516       | 0.519        |
| fresh best asym @ p_c ≥ 0.70      | −0.043      | −0.188 ⚠     |
| gate pass (AUC ≥ 0.63 AND asym ≤ −0.15) | ❌    | ❌           |

⚠ SmolLM's −0.188 asymmetry is **degenerate**: fresh positive class is 9/193 (4.7%). At threshold 0.06 the probe predicts "assert" on almost everything → `p_correct` trivially near 1.0, `p_wrong` marginally lower. AUC at chance (0.519) confirms there is no real direction; the asym number is a class-imbalance artifact, not a signal.

### Two distinct failure modes, one conclusion

- **Gemma** fails via *single output distribution*: every response is stylistically identical, so `hs_last` encodes "verbose confident essay about cooking," not truth-value. AUC ~chance at a healthy pos rate (0.285).
- **SmolLM** fails via *capability collapse*: fresh pos rate drops 12× from step16 (0.56 → 0.047), leaving no positive class for the probe to learn against. AUC ~chance at a tiny pos rate.

The two models fail `hs_last` probing for orthogonal reasons. The bottom line is the same: **at sub-2B instruct scale on judge-based domains, the frozen-base last-layer residual stream does not carry a content-editable correctness direction that a standard linear probe can read.** §11.6 closure is not a Gemma-instruct-tune quirk.

### Why the SmolLM disposition-axis sweep was skipped

- Factual anchor (the axis most likely to carry clean signal) failed on SmolLM same as Gemma.
- `fresh_pos = 9` has no statistical power to split into 4 disposition axes at ~5% base rate.
- v4 judge spend (~$1–2) was declined — the outcome is already determined and adds no generality to the record.

## 7. What's next (not a commitment — surfaced for user decision)

The user has an external recommendation on the table: stop iterating on surface methods (scalar head damping, hs_last sidecars, generic teacher SFT, style DPO) and do **execution-grounded process training for one functional trait in one verifiable domain**. Given the closure of §11.6, this is the natural candidate for the remaining DD v2 shot.

**Shape of the candidate (not a plan — a framing the user is evaluating):**

| Variable | Candidate | Why |
|---|---|---|
| Domain (must be verifiable) | **Code** (HumanEval + MBPP + LiveCodeBench-lite) | Only DD v2 target domain with a true execution oracle. Chef / GDPR / Companion are judge-based, therefore not verifiable in the required sense. |
| Base | Qwen3.5-0.8B (Coder) or Qwen3-1.7B | Must be able to write code at baseline. Gemma 4 E2B is not a code model. |
| Functional trait | **Persistence** (iterate on failing tests) or **Adversarial-self** (cross-check before submit) | Both are directly visible in an execution trajectory. Humble / Eager / Curious are not. Spot-check §4.5 identifies adversarial-self as Gemma's weakest axis, which would be a good disposition target if base were Gemma, but base is Qwen for verifiability reasons. |
| Signal | Execution-grounded trajectory SFT → GRPO with process reward | Cheap stage gated by whether it moves behavior without hurting pass@1. |
| Budget | Same $55 envelope saved from §11.6 GRPO | No new money. |
| Novelty hinge | SITS-style cross-domain transfer test | Without cross-domain transfer, this collapses to standard RLVR. The DD v2 contribution has to be that persistence trained on code transfers to reasoning / math / some other domain. |

**Hard prerequisites before committing any of this:**

1. Sandboxed executor on pod (planned in `project_coding_specialist.md`, never built).
2. Retry-trajectory dataset builder.
3. Trajectory scorer that rewards *both* final pass AND visible iteration behavior (not just pass@1, which is capability not disposition).
4. Baseline persistence measurement on Qwen3.5-0.8B-Coder (E0 step) — if baseline already persists >50%, the trait is not learnable on this base and we pick adversarial-self instead or change base.

**Explicit non-decision.** No target is locked. The user has said "wait for the run to finish" and "do not lock in anything". This section exists so the pivot discussion can pick up from the correct premise rather than relitigating §11.6.

## 9. Short version for a future reader

DD v2 §11.6 asked whether a frozen Gemma 4 E2B base's final hidden state carries enough of a correctness signal to gate hedging behavior via a small sidecar without retraining. Within-distribution CV at n=100 said yes (AUC 0.68, asym −0.34). Out-of-distribution on 193 fresh prompts from the same distribution said no, at every probe we tried: `hs_last` factual (AUC 0.52), `hs_last` on 4 dispositions (0/4 pass, 3 below chance), last-4 layer mean pool (AUC 0.56, asym −0.13). To rule out a Gemma-instruct-tune artifact, the factual anchor was re-run on SmolLM2-1.7B-Instruct: AUC 0.52, gate fail, by a different failure mode (fresh capability cliff to 4.7% pos rate). Two frozen bases, two distinct failure routes, one conclusion — **frozen-base `hs_last` at sub-2B instruct scale does not carry a content-editable correctness direction that a linear probe can read on judge-based domains**. The $55 GRPO follow-up was not authorized. The branch is closed. Behavior is real and the judge can read it; the frozen-base residual stream at the final token simply does not encode it. The candidate next branch — execution-grounded process training on a verifiable domain for one functional trait — is a different theory, not an extension of §11.6.
